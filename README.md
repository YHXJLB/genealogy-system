# 家谱系统 (Genealogy System)

## 项目概述

本项目实现了一个完整的中国古代名门望族家谱管理系统，采用**两种不同的数据结构**实现不同的功能需求。

### 应用背景

以中国古代著名的两大名门望族为背景：
- **孔子世家**：儒家文化的传承者
- **琅琊王氏**：书法世家的典范

## 数据结构设计

### 结构1：树型存储 + 链表 (Binary Search Tree)

**实现功能：**
1. ✅ 创建家谱
2. ✅ 查询家族成员信息
3. ✅ 查询所有祖先（至少五代）
5. ✅ 插入新成员

**特点：**
- 使用二叉搜索树存储家族成员
- 支持快速查询（O(log n)）
- 保存完整的族谱关系
- 包含历史坐标（字号、谥号、历史事件）

---

### 结构2：邻接表存储 (Adjacency List)

**实现功能：**
1. ✅ 创建家谱
2. ✅ 查询家族成员信息
4. ✅ 查询所有子孙（至少五代）
6. ✅ 删除成员（保留子孙）

**特点：**
- 使用哈希表 + 邻接表存储
- 高效的图遍历
- 支持灵活的删除操作
- 完整保存族谱的图结构

## 功能说明

### 通用数据结构

每个家族成员包含以下信息：
```
姓名 (name)              - 全名
字号 (courtesy_name)     - 古代成人名字
谥号 (posthumous_name)   - 身后追赠名号
朝代 (dynasty)           - 所在朝代
官职 (position)          - 最高官职
历史坐标 (coordinates)   - 历史事件/年份
生卒年 (birth_death)     - 生卒信息
成就 (achievement)       - 主要成就
```

### 功能1：创建家谱

- 初始化家谱系统
- 添加创始人（如孔子、王羲之等）
- 建立族谱关系

### 功能2：查询成员信息

输入：姓名/字号/谥号之一

输出：
- 完整个人信息
- 父亲信息
- 子女列表

```
查询示例：
输入："王羲之" 或 "逸少" 或 "书圣"

输出：
姓名：王羲之
字号：逸少
谥号：书圣
朝代：东晋
官职：右军将军
历史坐标：永和九年兰亭雅集-《兰亭集序》
生卒：303-361
成就：书法艺术的集大成者，"书圣"称号
父亲：王款
子女：王献之、王凝之等
```

### 功能3：查询祖先（至少五代）

- 从指定成员开始
- 递归查询所有父代
- 显示完整的上行族谱
- 至少显示5代祖先

### 功能4：查询子孙（至少五代）

- 从指定成员开始
- 递归查询所有子代
- 显示完整的下行族谱
- 至少显示5代子孙

### 功能5：插入新成员

- 指定父亲（必须是现有成员）
- 输入新成员的完整信息
- 自动建立族谱关系

### 功能6：删除成员

- 删除指定成员
- 保留其所有子孙
- 子孙自动继承新的父亲关系

## 技术特点

### 结构1（树型 + 链表）

**优势：**
- 快速成员查询（O(log n)）
- 高效的祖先追溯（直接向上遍历）
- 内存利用率高

**适用场景：**
- 频繁查询成员信息
- 需要快速祖先查询
- 族谱规模中等

### 结构2（邻接表）

**优势：**
- 灵活的图遍历
- 高效的删除操作
- 支持双向查询（祖先和子孙）
- 易于扩展

**适用场景：**
- 频繁删除操作
- 需要双向查询
- 复杂的族谱关系
